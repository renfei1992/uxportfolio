Axxess Web App for Homecare Agencies

My Role

In Summer 2016, I joined Axxess as a user experience design intern. One of my projects was to redesign companyâ€™s flagship product Agencycore focusing on info architecture and responsiveness. Agencycore is a web application aiming to empower healthcare agencies to provide the best care for their patients. I conducted usability test among nurses and caregivers and prototyped mockups from wireframes to high-fidelity designs.

Make the Hard Easy

AgencyCore is a complex product, just like any data management systems. It is complex in terms of both architecture and diversity of information. In order to define the goals of the redesign, we interviewed several users to understand who we are designing for. The following is a typical user with her motivations and goals:

1. Empower users to finish their tasks more efficient and navigate them with ease.

2. Enable users to access the web app using mobile devices without decreasing the quality of user experience.

Usability Studies and User Interviews

The first step to solve a problem is to realize there is one. In order to understand what is important to the context of healthcare, we interviewed several home health clinicians and conducted "think-aloud" usability tests.

UX Strategy and Framework

Mobile First

To keep consistent across all devices, we found lots of the interactions on desktop do not have equivalent interactions on mobile devices. For example, mobile devices do not have hover states while desktop uses very often; horizontal scroll is more natural on mobile than desktops. Therefore, we decided to go with the mobile-first design strategy.

Responsiveness and Grid System

We should never design for devices but always design for screen sizes. The web app was developed in 2007 without the help of a grid system, so now we need to adopt a 12-bar system that smoothly transits the UI across different screen sizes, no matter it's a 1440-pixel HD desktop screen or a 375-pixel iPhone 6 screen. We should cover them all.

Customizable Dashboard Card System

Using cards is maybe the hottest UX pattern in 2015, and we decide to utilize it as well. Instead of showing all information in tables, we used charts, icons and color labels to group and visualize information. Our design of the dashboard also allows rearrangement and customization.

A Bootstrap Structure

According to the web traffic analysis, a big portion of the users accesses the product via mobile devices. However, our current framework doesn't support responsiveness. We decided to embrace the Bootstrap grid system when redesigning so that users can always expect the same level of experience.

Master UI Patterns

Due to the complexity of the system, there are hundreds of pages in total waiting to be updated within three months. Facing these challenges, we decided to find the unique pages, categorize them, and adopt master UI patterns.

New Look, New Experience

We repainted the entire UI for 6 selected master views by replacing the skeuomorphic design with lightly weighted style in both visual and interaction.
